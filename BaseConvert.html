General Information

Universal Base Conversion Mechanics

Converting values between bases relies on positional numeral systems. Here are the core methodologies:

  • Positional Weighting (Expansion): Used primarily to convert any base to Decimal (Base 10). Each digit is multiplied by the base raised to its positional index: $$N = d_n \times B^n + d_{n-1} \times B^{n-1} \dots + d_0 \times B^0$$
  • Repeated Division (Modulo): Used to convert Decimal to any target base. You continuously divide the integer by the target base, extracting the remainder at each step. The sequence of remainders (read bottom-to-top) forms the new value.
  • Bit Grouping: Used exclusively for powers of 2 (e.g., Binary to Hexadecimal or Octal). Because $16 = 2^4$, every exactly four bits of binary map directly to one hexadecimal digit.

Human vs. Computer: For humans, Bit Grouping is by far the easiest method when moving between Binary, Octal, and Hexadecimal, as it requires zero arithmetic—only pattern matching. Conversely, computers do not "convert" bases internally; they process binary fundamentally. When forced to format outputs into strings (like decimal), computers exclusively use Repeated Division optimized via bitwise shifting and hardware modulo arithmetic.

Floating-Point Mathematics

Floating-point architecture is the computer's native implementation of scientific notation, prioritizing dynamic range over fixed precision. The generalized formula is: $$\text{Value} = (-1)^{\text{Sign}} \times \text{Significand} \times 2^{\text{Exponent} - \text{Bias}}$$

  • The Sign Bit: A single bit where $0$ is positive and $1$ is negative.
  • The Biased Exponent: Instead of using Two's Complement for negative exponents, standard hardware uses a "Bias." If the exponent uses $k$ bits, the bias is $2^{k-1} - 1$. This maps all possible exponents to positive unsigned integers, allowing the CPU hardware to sort and compare floating-point numbers faster using standard integer logic gates.
  • The Significand (Fraction): The normalized mantissa containing the significant digits of the value.

Architecture Standards & Examples

  • 32-bit IEEE 754 Standard (Implied 1.x): The global industry standard. Normalization forces the binary to be in the format $1.xxxxx$. Because the leading bit is always 1 (except for literal zero), hardware engineers drop it. This "Implied 1" grants 24 bits of mathematical precision while only physically storing 23 bits in RAM. Formula: $(-1)^S \times (1 + \text{Fraction}) \times 2^{E - 127}$
  • 32-bit Textbook Standard (No Implied 1, 0.1x): Common in academia to bridge the gap before introducing the Implied 1 hardware optimization. It normalizes numbers to fractions strictly between $0.5 \le M < 1.0$, meaning the binary always starts with $0.1$. It requires the full storage of the significand. Formula: $(-1)^S \times (0 + \text{Fraction}) \times 2^{E - \text{Bias}}$
  • 11-bit Classroom Example: A miniaturized architecture designed for manual, pencil-and-paper tracing. It utilizes 1 Sign bit, 5 Exponent bits, and 5 Fraction bits. To calculate the bias for a 5-bit exponent: $2^{5-1} - 1 = 15$. Formula: $(-1)^S \times (0 + \text{Fraction}) \times 2^{E - 15}$

How-To

Base Conversion

A "base" simply tells you how many unique digits exist before you have to carry over to the next column. Base 10 uses 0-9. Base 2 (Binary) uses 0-1. Base 16 (Hexadecimal) uses 0-9 and A-F. Here is how to manually translate between them:

  • Positional Weighting (Any Base → Decimal Base 10)
    Think of each position in a number as a bucket holding a specific "weight." The weight is the base raised to the power of its position (starting at 0 on the far right).
    Example: Convert Binary $1101_2$ to Decimal
    • Position 3: $1 \times 2^3 = 8$
    • Position 2: $1 \times 2^2 = 4$
    • Position 1: $0 \times 2^1 = 0$
    • Position 0: $1 \times 2^0 = 1$
    • Result: $8 + 4 + 0 + 1 = 13$
  • Repeated Division (Decimal Base 10 → Any Base)
    To pull a decimal number apart into another base, you continuously divide by your target base and keep the remainders. The remainders build your new number from right to left.
    Example: Convert Decimal $255$ to Hexadecimal (Base 16)
    • $255 \div 16 = 15$ with a remainder of $15$. In Hex, 15 is written as F.
    • $15 \div 16 = 0$ with a remainder of $15$. In Hex, 15 is written as F.
    • Result: Read remainders from bottom to top: FF
  • Bit Grouping (Binary ↔ Hexadecimal/Octal)
    Because computers process binary, moving between Binary and Hexadecimal is highly optimized. Hexadecimal (Base 16) is exactly $2^4$. This means exactly four binary bits perfectly represent one Hex letter. No math is required, just pattern matching.
    Example: Convert Hex $0x3A$ to Binary
    • $3 \rightarrow 0011$
    • $A \rightarrow 1010$ (A is 10 in decimal)
    • Result: $00111010$

Floating-Point Mathematics (The Concepts)

Computers cannot store infinite decimals. Instead, they use a form of scientific notation called Floating-Point. The generalized formula is: $$\text{Value} = (-1)^{\text{Sign}} \times \text{Significand} \times 2^{\text{Exponent} - \text{Bias}}$$

  • The Sign Bit: $0$ means positive. $1$ means negative.
  • The Biased Exponent: Hardware circuits struggle with negative numbers. Instead of making the exponent capable of being negative, engineers add a "Bias" (like 127) to the actual power. This forces all exponents to be stored as positive numbers, making it incredibly fast for the CPU to compare which number is larger.
  • The Significand (Fraction): This stores the actual digits of your number after it has been normalized (shifted so the decimal point is in a standard place).

How to Convert Floating-Point by Hand (IEEE 754 32-bit)

Decimal to IEEE 754 Binary:

  1. Find the Sign: If positive, bit is $0$. If negative, bit is $1$.
  2. Convert the absolute value to Binary: Whole numbers use standard division. For the decimal fraction, multiply by 2. The whole number part of the answer becomes your bit. Take the remaining fraction and repeat.
    Example for 0.84375:
    $0.84375 \times 2 = \mathbf{1}.6875 \rightarrow$ Bit is 1
    $0.6875 \times 2 = \mathbf{1}.375 \rightarrow$ Bit is 1
    $0.375 \times 2 = \mathbf{0}.75 \rightarrow$ Bit is 0
    $0.75 \times 2 = \mathbf{1}.5 \rightarrow$ Bit is 1
    $0.5 \times 2 = \mathbf{1}.0 \rightarrow$ Bit is 1. (Fraction is now 0, we stop). Binary: $0.11011$
  3. Normalize: Shift the decimal point until you get a number like $1.\text{something}$.
    $0.11011 \rightarrow 1.1011 \times 2^{-1}$. (We shifted right by 1, so the actual exponent is -1).
  4. Calculate Biased Exponent: Add the standard 32-bit bias (127) to your actual exponent. $-1 + 127 = 126$. Convert $126$ to 8-bit binary $\rightarrow 01111110$.
  5. Extract Significand (The Storage Hack): Because every normalized binary number starts with "1.", hardware designers don't bother storing it to save memory. Drop the leading 1, take the $.1011$, and pad it with zeros until it is 23 bits long: $10110000000000000000000$.
  6. Assemble: Put the Sign, Exponent, and Significand bits together.

IEEE 754 Binary to Decimal:

  1. Split the string: Break the 32 bits into a 1-bit Sign, 8-bit Exponent, and 23-bit Significand.
  2. Unbias Exponent: Convert the 8-bit exponent to decimal and subtract 127 to get your actual shift value.
  3. Restore the Implied 1: Put "1." back in front of your 23-bit Significand.
  4. De-normalize: Move the decimal point left or right based on the actual exponent you found in step 2.
  5. Convert back to Base 10: Use positional weighting on the resulting binary number to find your final decimal value, applying the negative sign if your Sign bit was 1.
