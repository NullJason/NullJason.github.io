The Big Picture: Macro-Architecture

A computer is a system of specialized components communicating across a motherboard. When you play a game or run a program, data flows constantly between them:

CPU (Central Processing Unit): The brain. It fetches instructions, performs calculations, and coordinates the rest of the system.
RAM (Random Access Memory): The short-term workspace. It holds the data and code the CPU is currently actively using.
GPU (Graphics Processing Unit): A specialized processor designed to handle rendering images and complex parallel math (like drawing a video game screen).
Storage (SSD/HDD): The long-term vault where files live when the computer is turned off.
Motherboard (Buses): The physical highway (data, address, and control buses) that allows all these components to send electrical signals to one another

What is an ISA?

ISA stands for Instruction Set Architecture. It is the bridge between the hardware (the physical silicon) and the software (your code). You can think of it as the CPU's vocabulary. It defines exactly what commands the processor understands (like ADD, LOAD, JUMP), how memory is accessed, and what registers are available. Two CPUs can have completely different internal physical designs (microarchitectures) but run the same software if they share the same ISA (like x86 or ARM). Another way to view it is that it is the SPECIFICATION of what you promise that your computer will do. It defines things such as the number of registers that will be available, and what each of the opcodes does.

Data Representation

Computers only understand electricity being ON (1) or OFF (0). Everything must be converted to binary.

Base Systems:: We use Base-10 (Decimal). Computers use Base-2 (Binary) and often display data in Base-16 (Hexadecimal) because it's easier for humans to read long binary strings (one Hex digit perfectly represents four binary bits).
IEEE 754 Floating-Point: How computers handle fractions and massive numbers. It splits a 32-bit (single precision) or 64-bit (double precision) sequence into three parts:
  1. Sign bit: 0 for positive, 1 for negative.
  2. Exponent: Shifts the decimal point.
  3. Significand (Mantissa/Fraction): The actual digits of the number.
The Implicit Leading 1: In standard normalized binary floating-point, the number always starts with a 1. Because it always starts with a 1, hardware designers decided not to waste a bit storing it. It is "hidden" or "implicit," giving us an extra bit of accuracy for free. The formula for evaluating this is:
(-1)sign × (1 + fraction) × 2exponent - bias
The converter offers 3 conversion formats, including no implicit 1.

The Micro-Level: Transistors & Logic Gates

  1. Transistors: In modern CMOS circuits, transistors act as electrically controlled gates. Applying a voltage opens or closes the path for current, creating 1s and 0s.
  2. Logic Gates: By wiring a few transistors together, we create Logic Gates—circuits that make simple decisions.
    • AND: Outputs 1 only if both inputs are 1.
    • OR: Outputs 1 if either input is 1.
    • NOT: Flips a 1 to a 0, or a 0 to a 1.
    • XOR (Exclusive OR): Outputs 1 only if the inputs are different.
  3. Pre-made Operations (Adders): By combining gates, we build math. For example, a "Half-Adder" connects an XOR gate and an AND gate to add two binary bits together and calculate a "carry" bit.

How to Use the CPU Simulator

Write Assembly: Open the Assembly Editor popup to write your program. You can also hover over the "Examples" button to load pre-written educational programs.
Configure Architecture: Adjust parameters like cache size, memory latency, and branch predictor type in the Configuration Panel.
Run & Step: Click "Assemble & Run" to compile your code. Use the playback controls (Play, Pause, Step Forward, Step Back) to move through the execution cycle by cycle.
Analyze: Watch the datapath visualizer on the left to see data flow, track the 5-stage pipeline at the bottom, and monitor performance graphs and memory states on the right.

Supported Assembly Instructions (Toy ISA)

Arithmetic/Logic
  • ADD rd, rs, rt (rd = rs + rt)
  • SUB rd, rs, rt (rd = rs - rt)
  • ADDI rd, rs, imm (rd = rs + imm)
  • ANDI rd, rs, imm (rd = rs & imm)
Memory
  • LD rd, [rs + imm] (Load word into rd from memory address rs + imm)
  • ST rs, [rd + imm] (Store word from rs into memory address rd + imm)
Control Flow
  • BEQ rs, rt, label (Branch to label if rs == rt)
  • J label (Unconditional jump)
System
  • NOP (No operation)
  • HALT (Stop execution)

The Classic 5-Stage RISC Pipeline

Pipelining improves CPU performance by allowing multiple instructions to be processed simultaneously, much like an assembly line in a factory. Instead of waiting for one instruction to finish entirely before starting the next, a pipelined CPU divides the work into stages:

IF
Instruction Fetch
Retrieves the next instruction from memory using the Program Counter (PC).
ID
Instruction Decode
Translates the instruction into control signals and reads the required values from the register file.
EX
Execute
The Arithmetic Logic Unit (ALU) performs mathematical operations, logical operations, or calculates memory addresses.
MEM
Memory Access
If the instruction is a Load or Store, the CPU reads from or writes to the data memory (or cache).
WB
Writeback
The final result is written back into the destination register.

Simulator Statistics Glossary

These counters help users connect what they see in the pipeline and memory views with real performance effects. They are especially useful when discussing hazards, stalls, and prediction behavior.

Cycles: The total number of clock cycles that have elapsed since execution started.
Instructions Retired: The number of instructions that have successfully completed all 5 stages of the pipeline.
Stalls (Bubbles): Cycles wasted because an instruction had to wait for a hazard to resolve (e.g., waiting for data from memory or an older instruction).
Branch Mispredictions: Occurs when the CPU guesses the wrong path for a branch (like an if statement). The pipeline must be "flushed" (emptied) of the wrong instructions, costing time. This is caused by unpredictable, random data or changing branch patterns which the branch prediction unit (BPU) in a CPU fails to see a pattern.
Cache Misses: Occurs when the CPU looks for data in the fast, temporary CPU Cache memory but doesn't find it, forcing it to wait to fetch the data from the slower Main Memory (DRAM).
